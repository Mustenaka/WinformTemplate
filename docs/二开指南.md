# 二开指南：按 DemoNote 模式新增模块

本文面向在 WinformTemplate 上继续加业务模块、改数据源、接后端的二次开发。目标是在约 30 分钟内加出一个可运行的最小模块，并保持 Phase II 的红线不破。

推荐先看两个完整样板：

- Product：常规业务模块，走 `UI -> ViewModel -> Service -> IRepository`，数据源由 `DataSource.Modules.Template` 切换。
- DemoNote：教学展示模块，同一个实体做 EF / WebAPI / Local 三个固定页面，用来对照三种数据源写法。

## 0. 开始前

先确认这些规则：

- 仍使用 .NET 8，不升 .NET 9。
- 仍使用项目自造 MVVM：`ObservableObject`、`BaseViewModel`、`RelayCommand`、绑定扩展。不要引入第三方 MVVM 框架。
- UI、ViewModel、Service 不引用 `DbContext`、`HttpClient`、EF 类型或 `Expression<Func<...>>`。
- 查询条件必须下推到 Repository；不要在 Service/ViewModel/UI 做 `GetAll().Where(...)`。
- 客户端和后端共享同一份 REST 契约：[api-contract.md](api-contract.md)。
- WebAPI 传输不可达必须抛 `DataSourceUnavailableException`，不能返回空集合伪装成功。
- 密码仍使用 PBKDF2；不要回退明文或弱哈希。

## 1. 新模块最小 6 步

下面用 `Order` 作为示例模块名。真实开发时替换为你的业务名。

### Step 1: Model

目录：

```text
WinformTemplate/Src/Business/Order/Model/
```

做法：

- 定义 `OrderModel`，用 EF attribute 标明表名和列名。
- 主键用 `long Id` 或模块内统一命名。
- 页面展示字段可加 `[NotMapped]`，不要把重复格式化逻辑散在 UI。
- 状态字段语义写清楚。Sys 模块中 `false` 表示有效、`true` 表示冻结或无效，这是历史约定。

参考：

- `WinformTemplate/Src/Business/Template/Model/ProductModel.cs`
- `WinformTemplate/Src/Business/Demo/Model/DemoNote.cs`

### Step 2: Repository Contract

目录：

```text
WinformTemplate/Src/Business/Order/Repositories/
```

示例：

```csharp
public interface IOrderRepository : IRepository<OrderModel>
{
    Task<PagedResult<OrderModel>> SearchOrdersAsync(
        string? keyword = null,
        int pageIndex = 1,
        int pageSize = 20,
        string? orderBy = null,
        bool ascending = true);
}
```

原则：

- 继承 `IRepository<T>`，保留 `GetByIdAsync`、`QueryAsync`、`AddAsync`、`UpdateAsync`、`DeleteAsync`。
- 复杂查询提供具名方法，例如 `IDemoNoteRepository.SearchByTitleAsync` 或 `IProductRepository.SearchProductsAsync`。
- 不暴露 `IQueryable` 或 `Expression<Func<...>>`，因为 WebAPI 和 Local 都无法共享 EF 表达式。
- 分页、关键字、排序、过滤统一进入仓储层。

### Step 3: 三种 Repository 实现

目录建议：

```text
Repositories/
  EfCore/EfOrderRepository.cs
  WebApi/ApiOrderRepository.cs
  Local/LocalOrderRepository.cs
```

通用注册在 `AppServiceRegistration.RegisterRepositories`：

```csharp
services.AddModuleRepository<IOrderRepository, EfOrderRepository, ApiOrderRepository, LocalOrderRepository>("Order");
```

这样普通业务模块可以由 `config.json -> DataSource.Modules.Order` 切换 `Ef` / `WebApi` / `Local`。DemoNote 三页是例外：它们为了教学展示固定注入 `EfDemoNoteRepository`、`ApiDemoNoteRepository`、`LocalDemoNoteRepository`，不走配置切换。

### Step 4: Service

目录：

```text
WinformTemplate/Src/Business/Order/Service/
```

Service 只依赖仓储接口：

```csharp
public sealed class OrderService : IOrderService
{
    private readonly IOrderRepository _orders;

    public OrderService(IOrderRepository orders)
    {
        _orders = orders;
    }

    public async Task<(List<OrderModel> Items, int TotalCount)> SearchOrdersAsync(
        string? keyword,
        int pageIndex,
        int pageSize,
        string? orderBy,
        bool ascending)
    {
        var result = await _orders.QueryAsync(new QueryRequest
        {
            Page = Math.Max(pageIndex, 1),
            PageSize = Math.Max(pageSize, 1),
            Keyword = keyword,
            SortBy = orderBy,
            Desc = !ascending
        });

        return (result.Items.ToList(), result.Total);
    }
}
```

不要在 Service 里拿全量数据再内存过滤。Product 的 `ProductService` 是常规模块参考；DemoNote 为了展示三种仓储，页面 ViewModel 直接依赖 `IDemoNoteRepository`，这不是所有模块都必须照搬。

### Step 5: ViewModel

目录：

```text
WinformTemplate/Src/Business/Order/ViewModel/
```

ViewModel 负责：

- `CurrentPage`、`PageSize`、`TotalCount`、`TotalPages`。
- `SearchKeyword`、过滤条件、排序字段。
- `LoadDataAsync`、`SearchAsync`、`GoToPageAsync`、`ApplySortAsync`。
- 新增/修改/删除成功后刷新当前页。
- 捕获 `DataSourceUnavailableException` 并设置可读 `StatusMessage`。

参考：

- `ProductManagementViewModel`
- `DemoNoteManagementViewModel`
- `AccountManagementViewModel` 的分页状态和上下页命令

### Step 6: UserControl + DI + 菜单

UI 目录：

```text
WinformTemplate/UI/Business/Order/
```

要求：

- 页面用 `Dock`、`Anchor`、`TableLayoutPanel`、`FlowLayoutPanel` 等布局容器，不要依赖构造期实际尺寸。
- AntdUI 表格、输入、按钮、分页控件尽量复用现有页面模式。
- 页面构造期不要设置依赖宽高的 `SplitterDistance` 或大 `Panel1MinSize/Panel2MinSize`。如果必须用 `SplitContainer`，延迟到 `HandleCreated` / `Load` / `SizeChanged` 后安全设置。
- UI 不直接访问仓储、DbContext 或 HttpClient。

DI 注册：

```csharp
services.AddScoped<IOrderService, OrderService>();
services.AddTransient<OrderManagementViewModel>();
services.AddTransient<OrderManagementControl>();
```

页面注册：

```csharp
pageRegistry.Register("/order/list", sp => sp.GetRequiredService<OrderManagementControl>());
```

菜单 seed 必须同步：

- EF：`SysDatabaseInitializer` 中新增 `SysMenuModel` 和 `SysRoleAuthModel`。
- Local：`Resources/MockData/sysMenus.json` 和 `Resources/MockData/sysRoleAuths.json`。
- 注册表：`PageRegistryDefaultPages.RegisterDefaultPages()`。

## 2. 加一个 EF 模块

以 DemoNote 的 EF 链路为模板。

### 2.1 文件结构

```text
WinformTemplate/Src/Business/Order/
  Context/
    OrderDbContext.cs
    OrderDbContextService.cs
    OrderDatabaseInitializer.cs
  Model/
    OrderModel.cs
  Repositories/
    IOrderRepository.cs
    EfCore/EfOrderRepository.cs
```

### 2.2 DbContext

参考：

- `DemoDbContext`
- `TemplateDbContext`
- `SysDbContext`

`OrderDbContext` 只包含 Order 模块自己的 `DbSet` 和模型配置。不要把多个模块塞进同一个 Context。

### 2.3 DbContextService

注册时使用模块 key：

```csharp
EfDbContextOptions.UseConfiguredDatabase(options, config?.Ef, "Order");
services.AddScoped<IDatabaseInitializer, OrderDatabaseInitializer>();
```

默认 SQLite 会解析到：

```text
Resources/Database/order.db
```

### 2.4 Initializer

`OrderDatabaseInitializer` 实现 `IDatabaseInitializer`：

- `ModuleKey => "Order"`
- `DataSourceKind => DataSourceKind.Ef`
- `InitializeAsync` 中调用 `Database.EnsureCreatedAsync`
- 空表时写入少量种子

### 2.5 EF Repository

继承 `EfRepositoryBase<OrderModel>`。查询必须在数据库端完成：

- `Where`
- `OrderBy` / `OrderByDescending`
- `Skip`
- `Take`

排序字段使用白名单，不要把任意字符串反射成 SQL 逻辑。

### 2.6 Startup Integration Test

新增 EF 模块后必须扩展 `AppStartupIntegrationTests`：

- 使用真实 `AppServiceRegistration.BuildServiceProvider`。
- 执行 `InitializeDatabaseAsync`。
- 断言 `EfDbContextOptions.ResolveSqlitePath(config.Ef.SQLitePath, "Order")` 文件存在。
- 从 DI 解析 `EfOrderRepository` 或 `IOrderService` 做一次分页查询。

这一步是为了防止多个 EF 模块同跑时撞库或漏建表。单独的 DbContext 单元测试不能替代真实启动序列测试。

## 3. 接一个 WebAPI 模块

以 DemoNote WebAPI 链路为模板。

### 3.1 先写契约

先更新 [api-contract.md](api-contract.md)，它是客户端和后端唯一 REST 契约来源。至少写清：

- 实体字段和 JSON 字段。
- `GET /api/{Module}/{resource}?page=&pageSize=&keyword=&sortBy=&desc=`。
- `GET /api/{Module}/{resource}/{id}`。
- `POST /api/{Module}/{resource}`。
- `PUT /api/{Module}/{resource}/{id}`。
- `DELETE /api/{Module}/{resource}/{id}`。
- `ApiResponse<T>` 包络。
- `isTransportError` 语义。

DemoNote 的实际端点是：

```text
GET    /api/Demo/notes
GET    /api/Demo/notes/{id}
POST   /api/Demo/notes
PUT    /api/Demo/notes/{id}
DELETE /api/Demo/notes/{id}
```

### 3.2 客户端 Repository

继承 `ApiRepositoryBase<OrderModel>`：

```csharp
public sealed class ApiOrderRepository : ApiRepositoryBase<OrderModel>, IOrderRepository
{
    public ApiOrderRepository(IWebApiClient client)
        : base(client, "Order", "orders")
    {
    }

    protected override object GetEntityId(OrderModel entity) => entity.Id;
}
```

具名查询用 `BuildQueryString` 组装契约参数。传输失败不要 catch 成空数据；基类会把 `isTransportError=true` 转成 `DataSourceUnavailableException`。

### 3.3 后端实现

后端独立项目位于：

```text
D:\Work\Code\CSharp\WinformTemplateServer
```

它是独立 git 仓库，.NET 8 ASP.NET Core Minimal API + EF Core + SQLite。启动：

```powershell
cd D:\Work\Code\CSharp\WinformTemplateServer
dotnet run --project src\WinformTemplateServer\WinformTemplateServer.csproj
```

默认地址：

- `https://localhost:5001`
- `http://localhost:5000`

客户端 `WebApi.BaseUrl` 默认对齐 `https://localhost:5001`。开发期 Swagger：

```text
https://localhost:5001/swagger
```

后端响应必须严格使用 `ApiResponse<T>`。业务失败使用 `success=false` 且 `isTransportError=false`；连接拒绝、DNS、超时这类传输失败是客户端侧产生的 `isTransportError=true`。

### 3.4 WebAPI 测试

客户端侧参考 `ApiRepositoryTests`：

- fake `IWebApiClient` 断言 URL 映射。
- 断言 CRUD 端点与 `api-contract.md` 一致。
- 断言 transport error 抛 `DataSourceUnavailableException`。

后端侧参考 `WinformTemplateServer.Tests`：

- `WebApplicationFactory` 覆盖 CRUD。
- 覆盖分页、关键字、排序。
- 覆盖校验失败和 not found。

## 4. 用 Local 自写数据

以 DemoNote Local 链路为模板。

### 4.1 MockData

新增：

```text
WinformTemplate/Resources/MockData/orders.json
```

JSON 字段建议与 .NET 属性名兼容；`LocalRepositoryBase` 反序列化时大小写不敏感。

### 4.2 Local Repository

继承 `LocalRepositoryBase<OrderModel>`：

```csharp
public sealed class LocalOrderRepository : LocalRepositoryBase<OrderModel>, IOrderRepository
{
    private const string SeedFile = "orders.json";

    public LocalOrderRepository() : base(SeedFile)
    {
    }

    public LocalOrderRepository(string seedRoot) : base(SeedFile, seedRoot)
    {
    }

    protected override object? GetEntityId(OrderModel entity) => entity.Id;

    protected override void SetEntityId(OrderModel entity, long id) => entity.Id = id;
}
```

注意：

- Local CRUD 只在内存集合生效，不回写 JSON。
- 搜索、排序、分页仍然在 Repository 内做。
- 测试里如果要改数据，使用临时 seedRoot 构造 repository，避免进程级缓存互相污染。

## 5. DemoNote 三页如何对照

DemoNote 是最适合看三种数据源差异的样例：

| Concern | EF 页 | WebAPI 页 | Local 页 |
| --- | --- | --- | --- |
| Page | `EfDemoNoteControl` | `ApiDemoNoteControl` | `LocalDemoNoteControl` |
| Repository | `EfDemoNoteRepository` | `ApiDemoNoteRepository` | `LocalDemoNoteRepository` |
| Storage | `demo.db` | `WinformTemplateServer` 自有 SQLite | `Resources/MockData/demoNotes.json` + 内存 |
| Query | LINQ 下推到 SQLite | `/api/Demo/notes?...` | 内存集合过滤分页 |
| Backend off | 不受影响 | 显示未连接后端 | 不受影响 |

三页共用 `DemoNoteControlBase` 和 `DemoNoteManagementViewModel`。如果你想做“同一页面随配置切换数据源”，参考 Product；如果你想做“教学展示每个数据源一页”，参考 DemoNote。

## 6. 踩坑 Checklist

提交前逐条检查：

- 菜单 key 三处一致：`SysDatabaseInitializer`、`Resources/MockData/sysMenus.json`、`PageRegistryDefaultPages`。跑 `SysMenuSeedUrls_AreConsistentAcrossEfLocalAndRegisteredPages`。
- 新 EF 模块有独立 SQLite 文件，不与 Sys/Template/Demo 共用库。跑 `AppStartupIntegrationTests`。
- 页面构造不依赖真实尺寸。跑 `PageConstructionSmokeTests`。
- WebAPI 客户端和后端都按 `docs/api-contract.md`；不要口头约定第二份契约。
- `DataSourceUnavailableException` 能到 ViewModel 并显示降级语义。WebAPI 后端关闭时不能崩。
- Service/ViewModel/UI 没有 `DbContext`、`HttpClient`、EF 类型、`Expression<Func<...>>`。
- 没有 `GetAll().Where(...)` 这种仓储外内存过滤。
- 密码相关仍走 `PasswordHasher` PBKDF2。
- 新 UI 用自造 MVVM 和绑定扩展，不引入第三方 MVVM。
- 运行期 SQLite 文件不入库；`.gitignore` 已忽略 `*.db`、`*.db-shm`、`*.db-wal`。
